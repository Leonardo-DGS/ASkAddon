Expressions

Inventory Name
Pattern:
  • name of inventory
  • inventory's name
  • name of[ inventory] %inventory%
Since: 1.0
Returns the name of inventory given, or if not supplied, returns the name of inventory in inventory click events.
Examples:
on inventory click:
broadcast "%name of inventory%"
Json Message
Pattern:
  • json [message] [(of|from)] %string%
Since: 1.0
Returns a json message from a string, that can be customized with other expressions about json, and can be sent to a player.
Examples:
set {_jsonmessage} to json of "I'm a json message"
Json Message Command
Pattern:
  • %jsonmessage% suggest %string%
  • %jsonmessage% run %string%
Since: 1.0
Returns a json message, that suggest or run a command when a player clicks on it, that can be further customized or sent to a player.
Examples:
json message of "I'm a json message" suggest "/examplecommand"
json message of "I'm a json message" run "/examplecommand"
json message of "I'm a json message" suggest "/examplecommand" run "/anothercommand"
Json Message URL
Pattern:
  • %jsonmessage% open [url] %string%
Since: 1.0
Returns a json message, that open an url to the player when it clicks on the message, that can be further customized or sent to a player.
Examples:
json message of "I'm a json message" open url "www.adf.ly/1lrgfA"
json message of "I'm a json message" open "www.adf.ly/1lrgls"
Json Message Tooltip
Pattern:
  • %jsonmessage% tooltip %string%
Since: 1.0
Returns a json message, that show a tooltip to the player when it hover on the message, that can be further customized or sent to a player.
Examples:
json message of "I'm a json message" tooltip "I'm a tooltip"
Json Message Append
Pattern:
  • %jsonmessage% (then|append) %string%
Since: 1.0
Appends a json message cleared from effects to another json message, that can be further customized or sent to a player.
Examples:
json of "I'm a json message" tooltip "I'm a tooltip" suggest "/examplecommand" append "I'm another json message" tooltip "I'm another tooltip"
New Boss Bar
Pattern:
  • new boss[ ]bar
Since: 1.0
Creates a new boss bar.
Examples:
set {_bossbar} to new boss bar
Boss Bar Colour
Pattern:
  • [boss][ ][bar] bar colo[u]r of %bossbar%
Since: 1.0
Returns the color of a bossbar. You can also set it.
Examples:
set {_bossbar} to new bossbar
set bar color of {_bossbar} to red
Flags of Boss Bar
Pattern:
  • list of flags of [boss][ ][bar] %bossbar%
Since: 1.0
Lists flags of a boss bar.
Examples:
set {_bossbar} to new bossbar
list of flags of bossbar {_bossbar}
Players of Boss Bar
Pattern:
  • list of players of [boss][ ][bar] %bossbar%
Since: 1.0
Returns a list of players of a boss bar.
Examples:
set {_bossbar} to new bossbar
list of players of bossbar {_bossbar}
Boss Bar Progress
Pattern:
  • [boss][ ][bar] progress of %bossbar%
Since: 1.0
Returns the progress of a bossbar, that can be changed. It's a number between 0 and 1.
Examples:
set {_bossbar} to new bossbar
set bar progress of {_bossbar} to 0.4
Serialized Boss Bar
Pattern:
  • seriali(z|s)ed [(contents|data)][ of][ boss][ ][bar] %bossbar%
Since: 1.0
Returns a string that contains data of the bossbar, and that can be reconverted to the original bossbar.
Examples:
set {_bossbar} to new bossbar
set {serializedbossbar} to serialized data of {_bossbar}
Boss Bar From Serialized
Pattern:
  • boss[ ]bar from data %string%
Since: 1.0
Returns the bossbar from a previously serialized bossbar in a string form.
Examples:
set {_bossbar} to boss bar from data {serializedbossbar}
Drops of Block
Pattern:
  • drops of [block] %block%
  • drops of [block] %block% with [item] %itemstack%
Since: 1.0
Returns potential drops of a placed block, optionally if broken with the specified item.
Examples:
drops of block placed-block
drops of block placed-block with iron pickaxe
Skull Owner
Pattern:
  • [the ]skull owner of %itemstack%
Since: 1.0
Returns the skull owner name of a skull item.
Examples:
set {_skullowner} to skull owner of player's item
Object In Ram
Pattern:
  • object [with name] %string% [saved] in ram
Since: 1.0
Gets an object by name previously saved in ram.
Examples:
save object player's name in ram "playername"
set {_playername} to object "playername" saved in ram
New Line
Pattern:
  • (new[ ]line|nl)
Since: 1.0
Simply returns the newline character.
Examples:
on command /examplecommand:
trigger:
send "Message%nl%Message in a new line"
Free Memory
Pattern:
  • free memory
Since: 1.0
Returns the free memory of the java virtual machine.
Examples:
set {_freememory} to free memory
Maximum Memory
Pattern:
  • max[imum] memory
Since: 1.0
Returns the maximum ram memory of the java virtual machine.
Examples:
set {_maximummemory} to maximum memory
Total Memory
Pattern:
  • total memory
Since: 1.0
Returns the total ram memory of the java virtual machine.
Examples:
set {_totalmemory} to total memory
Java Version
Pattern:
  • java version
Since: 1.0
Returns java version of the server.
Examples:
set {_javaversion} to java version
Ping of Player
Pattern:
  • ping of [player] %player%
Since: 1.0
Returns the ping of player.
Examples:
on command /ping:
trigger:
send "Your ping: %ping of player%ms"
Max Players Count
Pattern:
  • max players count
Since: 1.0
Returns the max player count of the server.
Examples:
on command /maxplayerscount:
trigger:
send "Max players count: %max players count%"
Loaded Chunks
Pattern:
  • [all] loaded chunks
  • loaded chunks of all worlds
  • loaded chunks of [world] %world%
Since: 1.0
Returns the loaded chunks of a world if specified, otherwise all worlds.
Examples:
loaded chunks of all worlds
loaded chunks of "Spawn"
Json Message Item Tooltip
Pattern:
  • %jsonmessage% (item[ ]tooltip|tooltip[ ]item) %itemstack%
Since: 1.1.1
Returns a json message, that show an item as tooltip to the player when it hover on the message, that can be further customized or sent to a player.
Examples:
json message of "I'm a json message" item tooltip stone
json message of "I'm a json message" tooltip item player's tool
PlaceholderAPI Placeholders
Pattern:
  • placeholderapi [parse[d]] place[ ]holder[s] [(from|with)] %string% [[(from|for)] %player%]
Since: 1.1.4
Parse PlaceholderAPI's placeholders from a string and returns the string with the placeholders replaced. You have to double percentages in these expressions.
Examples:
on command /testplaceholderping:
trigger:
set {_placeholderapiping} to placeholderapi parsed placeholders from "Your ping: %%player_ping%%" for player
message "%{_placeholderapiping}%"
